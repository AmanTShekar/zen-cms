# Zenith CMS — Code Examples & Integration Recipes

This document provides definitive implementation patterns for integrating external services with Zenith CMS via the Node.js SDK and webhook security layers.

---

## 1. Edge Runtime Data Fetching

When consuming the Zenith Core API from an edge runtime (e.g., Next.js Edge API routes, Cloudflare Workers), utilize the official SDK to handle tenant scoping and API token injection automatically.

```typescript
import { createZenithClient } from '@zenith-open/zenithcms-sdk'

const cms = createZenithClient({
  baseUrl: process.env.ZENITH_CORE_API_URL || 'http://localhost:3000/api/v1',
  apiKey: process.env.ZENITH_WORKSPACE_TOKEN,
  defaultSiteId: 'site_sandbox_901a',
})

/**
 * Executes an optimized document retrieval for rendering storefront pages.
 * Enforces a strict limit to minimize database overhead.
 */
export async function getStorefrontPage(slug: string) {
  try {
    const page = await cms.collections('pages').find({
      filter: {
        slug: slug,
        status: 'published',
      },
      limit: 1,
    })

    if (!page.docs.length) {
      throw new Error(`Page not found: ${slug}`)
    }

    return page.docs[0]
  } catch (error) {
    console.error('Data retrieval failed in getStorefrontPage:', error)
    return null
  }
}
```

## 2. Secure Webhook Verification (HMAC-SHA256)

Outbound webhooks generated by Zenith CMS are signed cryptographically. All external ingestion points must verify this signature using a constant-time comparison to mitigate timing attacks.

```typescript
import crypto from 'crypto'
import { Request, Response } from 'express'

const WEBHOOK_SIGNING_SECRET = process.env.ZENITH_WEBHOOK_SECRET

/**
 * Validates and processes incoming webhook events from Zenith CMS.
 */
export async function handleZenithWebhook(req: Request, res: Response) {
  if (!WEBHOOK_SIGNING_SECRET) {
    throw new Error('Server configuration error: Webhook secret is undefined.')
  }

  const signature = req.headers['x-zenith-signature'] as string

  if (!signature) {
    return res.status(401).json({ error: 'Signature header is missing.' })
  }

  const computedHash = crypto
    .createHmac('sha256', WEBHOOK_SIGNING_SECRET)
    .update(JSON.stringify(req.body))
    .digest('hex')

  const isValid = crypto.timingSafeEqual(
    Buffer.from(signature, 'hex'),
    Buffer.from(computedHash, 'hex')
  )

  if (!isValid) {
    return res.status(403).json({ error: 'Signature validation failed.' })
  }

  const { event, collection, data } = req.body
  console.log(`[Webhook Verified] Event: ${event} | Target: ${collection}`)

  // Acknowledge receipt to clear the queue
  return res.status(200).json({ received: true })
}
```

## 3. Server-Side Lifecycle Hooks

Lifecycle hooks execute on the backend Node.js process and can intercept database mutations. Use hooks to implement complex validation rules, state updates, or trigger third-party system synchronizations.

```typescript
import { CollectionConfig } from '@zenith-open/zenithcms-types'

export const OrdersCollection: CollectionConfig = {
  slug: 'orders',
  name: 'Orders Ledger',
  fields: [
    { name: 'orderId', type: 'text', required: true },
    { name: 'amount', type: 'number', required: true },
    { name: 'tax', type: 'number' },
    { name: 'status', type: 'text', defaultValue: 'pending' },
  ],
  hooks: {
    beforeChange: [
      async ({ data, operation }) => {
        // Enforce deterministic server-side calculations
        if (operation === 'create' || operation === 'update') {
          data.tax = Number((data.amount * 0.18).toFixed(2))
        }
        return data
      },
    ],
    afterChange: [
      async ({ doc, operation }) => {
        if (operation === 'create') {
          console.log(`[Invoice Trigger] Order ${doc.orderId} recorded successfully.`)
        }
      },
    ],
  },
}
```
